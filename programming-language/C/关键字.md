# 关键字

## 关键字列表
| 关键字 | 说明 |
| --- | --- |
| auto | 声明变量为自动类型 |
| const | 用于定义常量  |
| extern | 声明变量在其他文件中定义 |
| register | 声明变量为寄存器类型 |
| static | 声明变量为静态变量 |
| volatile | 声明变量易变，防止优化 |

### const    
const修饰的变量必须初始化 ，相当于声明为只读变量，不能修改。  
如 const int a = 10;  
   int const a = 10; //两者等价  
#### 易混淆  
const int *p;  // cosnt在*前面修饰 *p ,不能通过*p修饰指向空间的数据。    
int *const p = &a;  // cosnt在*后面修饰的是p，指针只能指向当前值。  
const int * const p = &a;  // p和*p都是只读的。  

### extern  
用来声明变量给外部文件使用，函数一般不需要  

### static   
1.全局：变量/函数只能在当前文件、模块里面使用  
2.局部：如代码段和函数段，变量/函数只能在当前函数里面使用，只用初始化一次  

### register
register 关键字用于建议编译器将变量存储在 CPU 寄存器中而不是内存中，以提高访问速度。register修饰的变量存放在寄存器中， register修饰的变量不能再使用指针访问。在现代编译器中，register 关键字的作用已经减弱，因为编译器的优化能力很强，通常能够自动识别需要优化的变量。因此，现在很少显式使用 register，主要依靠编译器自动优化。  

### volatile  
修饰的变量不优化，每次都要从内存中获取。可变的意思，修饰的变化会被改变，不让编译器做优化。
